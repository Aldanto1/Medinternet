# Деплой Medinternet_bot в облако

Mini app открывается на телефонах пользователей и требует HTTPS. Локальные
туннели в текущей сети душит DPI, поэтому бот вместе с веб-сервером
разворачивается на облачном хостинге.

## Что нужно от хостинга

- **Постоянно работающий процесс** (бот держит long-polling). Подходят Railway,
  Fly.io, обычный VPS. Бесплатный тариф Render засыпает при простое — для
  polling не годится без доработки на вебхуки.
- **Публичный HTTPS-домен** — платформа выдаёт его автоматически.
- **Открытый порт** — платформа передаёт его через переменную `PORT`
  (код уже это учитывает).

## Переменные окружения (задаются в панели хостинга, НЕ в файле)

| Переменная     | Значение                                                            |
|----------------|---------------------------------------------------------------------|
| `BOT_TOKEN`    | Токен бота от @BotFather                                             |
| `DATABASE_URL` | Строка подключения Neon (та же, что в `env/.env`)                    |
| `WEBAPP_URL`   | Публичный адрес сервиса, напр. `https://medinternet.up.railway.app` |

- `PORT` задаёт платформа сама — вручную не трогать.
- `PROXY_URL` в облаке **не нужен** (сеть чистая) — не задавать.
- `WEBAPP_URL` узнаётся только после первого деплоя (когда платформа выдаст
  домен). Схема: задеплоить → скопировать выданный URL → вписать в `WEBAPP_URL`
  → передеплоить.

## Пример: Railway (самый простой путь)

1. Залить код в репозиторий на GitHub.
2. На https://railway.app → **New Project → Deploy from GitHub repo** → выбрать репозиторий.
3. Railway увидит `Dockerfile` и соберёт образ автоматически.
4. Во вкладке **Variables** добавить `BOT_TOKEN` и `DATABASE_URL`.
5. Во вкладке **Settings → Networking → Generate Domain** — получить публичный URL.
6. Добавить переменную `WEBAPP_URL` = этот URL и дождаться передеплоя.
7. Открыть бота в Telegram, `/start` → кнопка «Регистрация» откроет mini app.

## Локальный запуск (для разработки)

```powershell
.\venv\Scripts\python.exe bot.py
```

Локально mini app по HTTPS не отдать без рабочего туннеля, но сам бот и команды
(`/start`, `/help`) работают. Полноценно форму регистрации проверяем на облаке.
